# ワークフロー・エージェント改善メモ

このドキュメントでは、AI映像制作ワークフロー全体および各サブエージェントの挙動に関する改善点、気づき、提案を記録します。

---

## レビュー: 2026-04-26 cupnoodle v1 完成版

`output/2.cupnoodle/cupnoodle_final.mp4` の完成版を視聴したフィードバック。

### 指摘事項

#### 1. BGM・歌がないため感情がのりにくい

- **現状**: `video-editing` 工程で無音マスターのみ書き出し、BGM/SEは「後乗せ推奨」として`editing.md`にコマンド例を記載するだけ
- **問題**: 視聴体験として感情のピークを支える音響の土台がなく、20秒の流れに起伏がつかない。特に C5（涙の extreme close-up）はBGMが持ち上がる前提で設計されたが、無音だと余韻が伝わらない
- **改善方針**:
  - `video-editing` の標準出力に BGM トラックを含める
  - BGM素材は (a) ユーザー持ち込み、(b) ロイヤリティフリーライブラリ参照、(c) AI音楽生成（Suno / Udio API）の3経路を選択可能にする
  - `editing.md` で「BGM の盛り上がりポイント」「カット秒数とBGMの拍合わせ」を必須記載項目とする

#### 2. エンドカードで「何の商品か」が伝わりにくい

- **現状**: C6-3 にカップヌードルのパッケージ＋メインコピー「お母さんの言葉の温度は、3分でやってくる」を焼き込んだが、コピーが情緒寄りで商品名・カテゴリ訴求が弱い
- **問題**: ブランディング目的のCMでも、最終カットで視聴者が「これはカップヌードルの広告だ」と即座に認識できないと購買行動に繋がらない。特にSNSの短時間視聴では1〜2秒で商品が伝わる必要がある
- **改善方針**:
  - エンドカード設計を **「メインコピー（情緒）」+「商品名/タグライン（識別）」の2層構造** にする
  - 例: 上層に「お母さんの言葉の温度は、3分でやってくる」、下層に「日清カップヌードル」「Nissin Cup Noodle」のロゴ＋商品名を明確に配置
  - コピー以外に「商品カテゴリ（即席カップ麺）」「容量」など必要に応じた識別情報も検討
  - `concept-design` の段階で「商品識別情報（ロゴ・商品名・タグライン）をエンドにどう配置するか」を必須項目に追加

#### 3. ナレーションの選択肢を持つ

- **現状**: cupnoodle はナレーションなし方針で `concept.md` に明記
- **問題**: 一律「ナレーションなし」をデフォルトにすると、訴求力を失うCMがある（特に親世代向け、商品説明が必要なケース、感動系の母の声などの演出）
- **改善方針**:
  - `concept-design` の音響設計を「ナレーション要否」を必ず判定するフェーズにする
  - 判定ガイドライン:
    - 動画だけで訴求が完結する場合 → ナレーションなし（無音 + BGM/SE）
    - 親世代訴求 / 商品説明が必要 / 感動系で声の温度が必要な場合 → ナレーションあり
  - ナレーション「あり」の場合は `storyboard` で各カットのセリフ・タイミング・話者像（例: 落ち着いた母性的な40代女性）を定義
  - 実装手段: AI音声合成（ElevenLabs / OpenAI TTS / VOICEVOX 等）を `scripts/generate-narration.ts` 等で標準化

## エージェント別の改善点

### concept-design

- [ ] **音響設計フェーズの追加**: ナレーション要否、BGMの方向性（ジャンル / テンポ / 楽器）、SEレイヤーをコンセプト段階で明文化する
- [ ] **エンドカード設計の必須項目化**: メインコピー（情緒）に加えて、商品名・ロゴ・タグライン・CTAを必ず記録する。情緒コピーだけで終わらせない
- [ ] **ターゲット別のナレーション判定ガイドライン**: 親世代訴求は声あり推奨、Z世代向けTikTokは声なし推奨、等の判断基準を定義

### storyboard

- [ ] **ナレーション/セリフの記述欄を必須化**: 各カットでセリフがある場合、タイミング・話者像・読み上げトーンを明記する
- [ ] **エンドカード（最終カット）の構成要素を分解**: メインコピー / 商品名 / ロゴ / タグライン / 余白配置 を個別に記述する欄を設ける
- [ ] **BGMの盛り上がりポイント**: 各カットの感情曲線とBGMの音量/楽器構成の対応を記述する

### asset-generation

- [ ] **エンドカード生成プロンプトのテンプレート化**: 「メインコピー（上層）+ 商品名/ロゴ（下層）」の2層構造を標準パターンとして提供
- [ ] **商品識別情報の確実な描画**: 商品名（日本語/英語）、ロゴ、タグラインの正確な描画を `quality=high` + `preserve text exactly` で確実化

### video-generation

- [ ] 大きな改善点はなし（Veo 3.1 Lite 採用継続、人物は日本人ルール、テキスト保護指示は既に標準化済み）
- [ ] 必要に応じて、ナレーションありCMでは口の動きとリップシンク程度の配慮を追加

### video-editing

- [ ] **BGMトラック付きマスターの標準化**: 無音書き出しではなく、BGMが乗った状態をデフォルト出力とする
- [ ] **音響レイヤーの具体的な実装手順**: BGM / SE / ナレーションのミキシング手順を `edit.sh` に組み込む
- [ ] **`scripts/edit-video.ts` 共通化案との統合**: BGM入力・ナレーション入力をconfig側で受け取り、ffmpegで統合する設計
- [ ] **ラウドネス基準（-14 LUFS など）への自動正規化**: 配信プラットフォーム要件への準拠

## クリエイティブ・ルールの更新案

### 追加候補

- **エンドカードは「情緒コピー + 商品識別情報」の2層構造を必須とする**
  - 情緒コピー単独では商品認知が弱く、ブランディング目的でも購買想起を欠く
  - 識別情報: 商品名（日本語＋英語）/ ブランドロゴ / 必要に応じてカテゴリ・容量

- **音響設計を `concept-design` 段階で必ず定義する**
  - ナレーションの有無と方針
  - BGMの方向性（ジャンル / テンポ / 楽器構成）
  - SEレイヤーの設計

- **ナレーション要否の判定基準**
  - 動画のみで訴求が完結 → ナレーションなし
  - 親世代訴求 / 商品説明 / 感動系の声の温度 → ナレーションあり
  - ハイブリッド（コピー直前の母の囁きなど、最小限の声）も選択肢として保持

### 既存ルール（参考）

- 登場人物は全員日本人として描写（例外指定がない限り）
- エンドカードはキャッチコピー焼き込み必須（GPT Image 2 quality=high で日本語焼き込み）
- 親オーケストレータがユーザー対話、サブエージェントはファイル生成（AskUserQuestion 制約）
